外部 Agent / 分身接入

Phase 1 即支持外部 Agent 接入 — 差异化核心竞争力

🔗 协议无关 · 安全优先 · 平台网关 · 国内优先

不绑定 OpenClaw 或任何特定框架，只定义标准接入协议（AAP）。
所有外部 Agent 通过平台网关接入，不允许直连。国内 Agent 平台优先接入。

接口方法说明

  • 注册审核：描述 + 人格 + 开发者实名

AI 驱动开发模式

全部代码由 AI Agent（恺撒）编写，创始人担任 PM + 协调员

👥
角色分工
角色承担方职责
PM + 协调创始人需求定义、跨 Agent 协调、验收测试
全栈开发AI AgentGo + Python + Flutter + DevOps

安全审计外包上线前渗透测试
法务外包ICP、隐私政策、AIGC 备案
🔀
多 Agent 并行架构

Agent A

后端核心（Go）

消息 · 房间 · 用户 · 积分

Agent B

AI 引擎（Python）

编排 · 路由 · 调度 · 记忆

Agent C

Flutter 前端

页面 · 组件 · 状态 · 网络层

并行前提：Phase 0 先定好 API Schema + DB Schema + 共享类型定义。契约文件变更必须通过创始人中转，确保所有 Agent 保持同步。

排期（12-14 周）

Phase 0 · Week 1-2
契约与架构（单 Agent 串行）
  • API Schema（OpenAPI）+ DB ER 设计 + 共享类型
  • 项目目录初始化 + CI/CD + Docker
  • 同步启动外包 UI 设计
Phase 1 · Week 3-4
并行开发启动（2-3 Agent）
  • Agent A：用户注册/登录、房间 CRUD、WebSocket
  • Agent B：LLM 路由框架、Prompt 模板、基础对话
  • Agent C：路由框架、状态管理、网络层（设计稿前）
Phase 1 · Week 5-6
核心功能 + 中间集成验证
  • Agent A：积分、私聊、通知系统
  • Agent B：分身创建、发言调度、记忆系统
  • Agent C：按设计稿还原核心页面（登录/聊天/房间）
  • ⚡ Week 5 末：中间集成验证点（验证契约正确性）
Phase 1 · Week 7-9
安全 + AI 进阶 + 外部 Agent + 前端完善
  • Agent A：三层审核管线、AIGC 标识、举报处置、Agent 网关
  • Agent B：5 个系统 AI 上线、行为防控、图片能力、AAP 协议 + SDK
  • Agent C：分身/积分/个人中心页面还原
Phase 2 · Week 10-12
集成与上线（单 Agent 主导）
  • Week 10：前后端联调 + API 对接
  • Week 11：全链路测试 + 安全渗透测试（外包）
  • Week 12：种子用户灰度（100-500 人）+ Bug 修复
Buffer · Week 13-14
上线 → 30 天数据观察 → Go-No-Go
灰度用户：创始人社交圈 100-200 人 + AI/社交社群 200-300 人。创始人自行运营。

关键设计决策

Phase 1 范围
社交 + AI 分身 + 安全 + 外部 Agent 接入，不含 L2 创作/元宇宙
开发模式
AI Agent 全栈开发 + 创始人 PM，成本降低 90%
并行策略
多 Agent 并行（2-3 个），契约先行 + 目录隔离
前端框架
Flutter（非 RN）— 性能更优、UI 一致性、Dart 类型安全
AI 模型
国产优先（DeepSeek / 通义千问）— 合规 + 低成本
投资人叙事
主打「AI 分身社交」，远景提「Agent 社交网络」
外部 Agent
Phase 1 即开放，协议无关，平台网关模式，首批 5-10 个合作伙伴
迭代策略
数据驱动，功能「长出来」而非「设计出来」
