# AI × 元宇宙社交平台 — Phase 1 MVP 需求文档（概要设计）

> **文档版本**：v1.2  
> **日期**：2026-03-15  
> **范围**：Phase 1 MVP（稳健方案 A：8-10 周）  
> **关联文档**：《AI×元宇宙社交平台：可行性调研与落地方案》

---

## 一、文档说明

### 1.1 MVP 范围定义

本文档覆盖 Phase 1 MVP 的全部功能，遵循可行性方案中"必须做"清单：

| 模块 | 范围 |
|---|---|
| **用户端** | |
| 用户系统 | 注册/登录（DID 绑定）、个人资料、设置 |
| 聊天系统 | 房间群聊、私聊、消息历史、审计回放 |
| 智能体系统 | 创建/配置 AI 分身、L1 发言、受控 L2 创作任务领取 |
| 内容标识 | AIGC 显式/隐式标识（生成、发布、转发链路） |
| 风控审核 | 内容审核、账号风控、智能体行为防控 |
| 积分计费 | 充值、消耗、账单、限额 |
| 冷启动 | 系统智能体暖场、新用户引导、种子内容 |
| **管理后台** | |
| 用户管理 | 用户列表/搜索/详情/封禁/解封、行为日志 |
| 智能体管理 | 系统智能体配置、用户智能体审核/下线、人格模板库 |
| 房间与话题管理 | 房间 CRUD、话题/创作任务发布与管理 |
| AI 网关管理 | 多模型配置/路由策略、API Key 管理、用量监控/预算告警 |
| 安全与审核管理 | 人审工作台、敏感词库管理、风控规则配置、处置记录 |
| 数据看板 | 核心指标监控（DAU/留存/发言率）、成本监控、Go-No-Go 仪表盘 |

### 1.2 明确不做

- 跨房间自主协作 / 工具调用 / 长链任务自治
- 可提现、点对点转账、类金融化能力
- 智能体家庭/社会关系等高敏功能
- 创作过程可视化回放（前端 UI 不做；但创作过程元数据——提示词、参数、时间戳——必须落库，见 §3.3）
- 竞赛系统（Phase 3）

## 二、用户角色定义

| 角色 | 说明 |
|---|---|
| **普通用户** | 注册并实名认证的真人用户，可聊天、创建智能体、参与创作 |
| **AI 分身（用户智能体）** | 用户创建的 AI 角色，可在房间内发言、领取创作任务 |
| **系统智能体** | 平台预设的 AI 角色，用于暖场、引导、话题发起 |
| **管理员/审核员** | 后台管理人员，负责内容审核、用户管理、系统配置 |

### 模块 1：用户系统

#### 1.1 注册与登录

**用户操作流程：**

```
手机号输入 → 获取验证码 → 填写验证码 → 设置密码
→ 实名认证（DID 绑定） → 创建昵称/头像 → 进入首页
```

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| U-01 | 手机号注册 | 国内手机号 + 短信验证码 | P0 |
| U-02 | 密码登录 | 手机号 + 密码 | P0 |
| U-03 | 验证码登录 | 手机号 + 短信验证码（免密登录） | P0 |
| U-04 | DID 绑定 | 注册完成后自动触发 DID 创建/绑定（后台静默完成） | P0 |
| U-05 | 忘记密码 | 手机号验证 → 重置密码 | P1 |
| U-06 | 退出登录 | 清除本地 Token，返回登录页 | P0 |

**业务规则：**

- 同一手机号仅允许注册一个账号
- 密码要求：8-20 位，必须包含字母和数字
- 短信验证码有效期 5 分钟，每分钟限发 1 条，每日限发 10 条
- DID 绑定失败不阻塞注册流程，后台异步重试

#### 1.2 个人资料

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| U-07 | 编辑昵称 | 2-16 字符，支持中英文/数字/emoji，需过敏感词 | P0 |
| U-08 | 编辑头像 | 上传图片或从预设头像选择，需过图片审核 | P0 |
| U-09 | 个人简介 | 可选填，最多 200 字，需过敏感词 | P1 |
| U-10 | 查看个人主页 | 展示昵称、头像、简介、创建的智能体列表 | P1 |

### 模块 2：聊天系统

#### 2.1 房间（主题房/兴趣房）

**用户操作流程：**

```
首页 → 浏览房间列表 → 选择房间 → 进入房间
→ 查看历史消息 → 发送消息 / 与智能体互动 → 离开房间
```

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| C-01 | 房间列表 | 展示所有公共房间，支持按分类/热度排序 | P0 |
| C-02 | 进入房间 | 点击房间卡片进入，显示房间信息与在线成员 | P0 |
| C-03 | 发送文本消息 | 在房间内发送文本消息，实时展示 | P0 |
| C-04 | 发送图片消息 | 支持发送图片（上传后需过审核） | P1 |
| C-05 | 消息实时推送 | WebSocket 实时推送新消息 | P0 |
| C-06 | 历史消息加载 | 向上滚动加载历史消息（分页，每页 50 条） | P0 |
| C-07 | @提及 | 在房间内 @某用户或智能体，被@方收到提醒 | P1 |
| C-08 | 房间信息查看 | 查看房间名称、描述、成员列表、当前话题 | P0 |
| C-09 | 房间搜索 | 按房间名称/标签搜索 | P1 |
| C-10 | 系统智能体标识 | 智能体发言旁永久展示"AI"标识，不可隐藏 | P0 |

**业务规则：**

- Phase 1 不支持用户创建房间，所有房间由管理员后台创建
- 单条文本消息限制 500 字
- 图片大小限制 5MB，支持 jpg/png/webp
- 房间限频：同一用户每分钟最多发 10 条消息
- 每个房间至少配置 2-3 个系统智能体（暖场要求）
- 单房间最大在线人数（含智能体）：200 人，达到上限后新用户排队等待

**异常处理：**

| 异常场景 | 处理策略 |
|---|---|
| WebSocket 断连 | 客户端自动重连（指数退避，最大间隔 30s），重连后拉取断连期间消息 |
| 消息发送失败 | 客户端展示发送失败状态，支持点击重发，3 次重试仍失败提示"网络异常" |
| 离线消息 | 用户上线后拉取离线期间未读消息（最多 200 条/房间），超出部分提示"查看更多" |

### 模块 3：智能体系统

#### 3.1 创建智能体（AI 分身）

**用户操作流程：**

```
个人主页 → "创建 AI 分身" → 填写基本信息
→ 设置角色人格 → 选择能力等级 → 确认创建
→ 消耗积分 → 创建完成
```

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| A-01 | 创建智能体 | 填写名称、头像、简介 | P0 |
| A-02 | 角色设定 | 设置人格（性格/口吻/特长）、行为边界、禁止事项 | P0 |
| A-03 | 能力等级选择 | L1（仅发言）或 L2（可领取创作任务），默认 L1 | P0 |
| A-04 | 智能体数量限制 | 每个用户最多创建 3 个智能体 | P0 |
| A-05 | 智能体列表 | 在个人主页展示已创建的智能体 | P0 |
| A-06 | 编辑智能体 | 修改名称、头像、人格设定 | P0 |
| A-07 | 删除智能体 | 删除智能体及其记忆数据，需二次确认 | P0 |
| A-08 | 积分消耗 | 创建智能体消耗固定积分（建议 50 积分） | P0 |

**业务规则：**

- 智能体名称 2-12 字符，需过敏感词
- 角色设定最多 500 字
- 角色设定中不允许包含引导转账、涉政、违法等内容
- 创建后智能体立即可用

#### 3.2 智能体 L1 — 房间发言

**用户操作流程：**

```
进入房间 → 选择"让我的 AI 分身发言" → 选择智能体
→ 智能体根据角色设定自主参与讨论 → 用户可随时暂停/恢复
```

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| A-09 | 智能体进入房间 | 用户指派智能体进入指定房间 | P0 |
| A-10 | 自主发言 | 智能体根据房间上下文和角色设定自动回复 | P0 |
| A-11 | 暂停/恢复 | 用户可随时暂停或恢复智能体的发言 | P0 |
| A-12 | 退出房间 | 用户可让智能体退出房间 | P0 |
| A-13 | 发言标识 | 智能体发言旁永久展示"AI"标识 + 创建者昵称 | P0 |
| A-14 | 发言限频 | 每个智能体每分钟最多发言 3 次 | P0 |
| A-15 | 每回合积分消耗 | 每次智能体发言消耗 2-3 积分 | P0 |

**触发策略：**

| 策略项 | 规则 |
|---|---|
| 触发条件 | 被 @时立即响应；未被@时，按"监听模式"——每 3 条新消息评估一次是否回复（由模型判断相关性） |
| 上下文窗口 | 取最近 20 条消息（约 2000 tokens）作为上下文，加上角色设定和记忆摘要 |
| 多智能体冲突 | 同一房间多个智能体同时触发时，按创建时间先后排队，相邻两个智能体发言间隔 >= 5 秒 |
| 发言频率 | 非被@场景：同一智能体最少间隔 30 秒发言一次，避免刷屏 |
| **空转评估成本** | **监听模式下，每次"是否回复"判断消耗约 2000 input tokens（不消耗用户积分，由平台承担）。热闹房间按 60 条/分钟、5 个智能体估算：约 100 次评估/分钟 ≈ 20 万 tokens/分钟。需纳入 AI 网关日预算管控（§11.2 M-35）** |

**业务规则：**

- 一个智能体同时只能在一个房间内活跃
- 智能体发言内容同样需过内容审核
- 智能体不得主动声称自己是真人
- 触发"冒充真人"检测规则时，直接拦截并通知创建者
- 智能体调用需设置幂等键（request_id），防止重复扣费和重复发言
- 模型调用超时（>10s）时触发熔断，该智能体暂停 60 秒后自动恢复
- 人格一致性：每次发言前注入角色设定 Prompt 模板 + 记忆摘要，确保跨会话人格一致（参见可行性方案附录 D5）

**异常处理：**

| 异常场景 | 处理策略 |
|---|---|
| AI 模型超时/不可用 | 该智能体自动暂停，房间展示"[智能体名称] 暂时休息中"，60 秒后自动重试 |
| 模型返回空内容 | 丢弃本次响应，不展示不扣费，等待下一次触发 |
| 发言被审核拦截 | 不展示该消息，向智能体创建者发送站内通知，该智能体限速 1 次/分钟 |
| 积分不足 | 智能体自动暂停，向用户发送"积分不足"站内通知 |

#### 3.3 智能体 L2 — 创作任务领取（受控模式）

**用户操作流程：**

```
话题广场 → 查看当前话题/创作任务 → 指派智能体领取任务
→ 智能体根据话题生成作品（图片） → 作品进入审核队列
→ 审核通过后发布到话题下
```

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| A-16 | 查看创作任务 | 展示平台发布的话题/创作任务列表 | P0 |
| A-17 | 领取任务 | 指派一个 L2 智能体领取创作任务 | P0 |
| A-18 | 生成作品 | 智能体根据话题关键词生成图片作品（单张） | P0 |
| A-19 | 作品审核 | 生成的作品自动进入审核队列，审核通过后发布 | P0 |
| A-20 | 作品展示 | 在话题详情页展示所有已审核通过的作品 | P0 |
| A-21 | 创作积分消耗 | 每次生成图片消耗 20-30 积分 | P0 |
| A-22 | 创作限额 | 每个智能体每日最多领取 3 个创作任务 | P0 |

**业务规则：**

- Phase 1 仅支持单张图片生成（1024x1024）
- 每个话题有截止时间，超时未完成自动释放，并向用户发送通知
- 作品生成后自动写入 AIGC 显式/隐式标识
- 创作任务由管理员后台发布
- 创作过程数据（提示词、参数、模型版本、时间戳）必须落库，为后续版权举证留数据基础（即使 Phase 1 不做可视化回放）

**异常处理：**

| 异常场景 | 处理策略 |
|---|---|
| 图片生成失败（模型错误） | 自动重试 1 次，仍失败则通知用户"生成失败，积分已退回"，积分自动回退 |
| 生成图片被审核拦截 | 通知用户"作品未通过审核"，积分不退（因模型资源已消耗），但不计入每日限额 |
| 任务超时未完成 | 自动释放任务，向用户发送站内通知，积分不扣（预扣积分自动回退） |

**功能点：**

### 模块 5：风控与审核

#### 5.1 内容审核

**处理流程：**

```
用户/智能体发送消息 → 同步审核（高危关键词匹配）
  ├─ 命中高危 → 直接拦截，记录证据，通知发送者
  ├─ 命中疑似 → 先发后审，进入异步审核队列
  │    ├─ 审核通过 → 保持展示
  │    └─ 审核不通过 → 下架/限流，通知发送者
  └─ 未命中 → 正常展示
```

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| R-01 | 文本敏感词检测 | 涉政、暴恐、色情、网暴、违法交易、引导转账 | P0 |
| R-02 | 对抗规避检测 | 谐音、拆字、符号替换、拼音变体 | P0 |
| R-03 | 图片审核 | 上传/生成图片过色情、暴恐、违法内容检测 | P0 |
| R-04 | 同步拦截 | 高危命中直接拦截，用户无法发送 | P0 |
| R-05 | 异步下架 | 疑似内容先展示，审核后不通过则下架 | P0 |
| R-06 | 人工审核台 | 管理后台的人工审核界面，支持通过/拒绝/标记 | P0 |
| R-07 | 用户申诉 | 被拦截的用户可提交申诉，进入人审复核 | P1 |

#### 5.2 账号风控

**功能点：**

| # | 功能 | 说明 | 优先级 |
|---|---|---|---|
| R-08 | 注册异常检测 | 同一设备/IP 短时间多次注册 → 触发验证码/封禁 | P0 |

#### 6.3 积分事务模型

所有积分消耗采用 **"预扣 → 执行 → 确认/回退"** 模式：

```
1. 预扣：冻结用户对应积分（余额减少，冻结池增加）
2. 执行：调用 AI 模型执行任务
3a. 成功：确认扣费（冻结池扣除）
3b. 失败：回退积分（冻结池退回余额）
```

**业务规则：**

- 预扣失败（余额不足）→ 直接拒绝请求，提示充值
- 并发请求：通过预扣冻结机制防止超发，冻结中的积分不参与余额可用判断
- 预扣超时（执行超过 60 秒未确认/回退）→ 自动回退，记录异常日志
- 每日对账：系统每日凌晨自动比对冻结池，超过 24 小时未处理的冻结自动回退

**异常处理：**

| 异常场景 | 处理策略 |
|---|---|
| 支付回调超时 | 主动查询支付状态，30 分钟内未确认则标记"待对账" |
| 重复支付回调 | 基于订单号幂等处理，重复回调直接返回成功，不重复加积分 |
| 充值成功但积分未到账 | 每 10 分钟对账任务扫描，差异自动补发并告警 |
| 积分扣了但生成失败 | 预扣模式保证：执行失败自动回退冻结积分 |

#### 6.4 降级策略

| 触发条件 | 降级动作 |
|---|---|
| 用户积分耗尽 | 仅保留免费额度，超出部分提示充值 |
| 平台日预算超阈值 | 自动切换低成本模型 |
| 单用户日消耗超限额 | 该用户进入冷却期至次日 |

## 五、非功能性需求

### 5.1 性能要求

| 指标 | 目标 |
|---|---|
| 消息发送到展示延迟 | <= 500ms（P95） |
| 页面首屏加载 | <= 2s（4G 网络） |
| 图片生成耗时 | <= 30s（P95） |
| API 响应时间 | <= 200ms（P95，不含 AI 调用） |
| 并发用户支持 | >= 1000 同时在线 |

### 5.2 安全与隐私保护

| 要求 | 说明 |
|---|---|
| 传输加密 | 全链路 HTTPS |
| Token 安全 | JWT + 2 小时过期 + Refresh Token |
| API 限流 | 按用户/IP 维度限流 |
| 敏感数据存储 | 手机号 AES 加密存储，密码 bcrypt 哈希，日志中手机号脱敏（138****1234） |
| 审计保留 | 所有消息 + 审核记录保留 >= 6 个月 |
| 数据最小化 | 只采集业务必需的用户数据，不采集通讯录、位置等无关信息 |
| 私聊内容保护 | 私聊内容仅双方可见，审核员仅在触发风控规则时可查看（需记录查看日志） |
| 数据导出 | 用户有权导出个人数据（待 Phase 2 实现） |

### 5.3 合规要求

| 要求 | 说明 |
|---|---|
| AIGC 标识 | 覆盖生成、发布、转发全链路 |
| 内容审核 SLA | P95 高危处置 <= 30 分钟 |
| 留痕字段 | 内容 ID、用户/智能体 ID、时间戳、审核结果码、处置动作 |
| 隐私协议 | 用户注册时需同意隐私政策和用户协议 |

## 六、验收标准（对应可行性方案 10.1 节）

> **说明**：“目标值”是团队努力方向，“Go-No-Go 底线”是决策门槛。验收标准未达标不等于 No-Go，但需分析原因并优化。

| 指标 | 目标值 | Go-No-Go 底线 | 检查周期 |
|---|---:|---:|---|
| D1 留存 | >= 20% | >= 15% | 7/14/30 天 |
| D7 留存 | >= 8% | >= 5% | 14/30 天 |
| 日均发言率 | >= 25% | >= 20% | 7/14 天 |
| 房间活跃深度 | >= 8 条/人/日 | >= 5 条/人/日 | 7/14 天 |
| 智能体使用率 | >= 10% | >= 5% | 7/14 天 |
| 事故 SLA | <= 30 分钟 | <= 60 分钟 | 7/14 天 |

**Go-No-Go 决策线（上线 30 天）：**

| 决策 | 条件 |
|---|---|
| ✅ **Go** | 底线全部达标且无重大合规事故 |
| ⚠️ **Pivot** | 1-2 项底线未达标但趋势向好 → 分析优化 + 延长 30 天 |
| ❌ **No-Go** | 3 项以上底线未达标或重大合规事故 → 暂停推广，重评定位 |
